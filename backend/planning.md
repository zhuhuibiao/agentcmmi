# 项目策划 (PMC)

## 用途
任务分解、依赖图、开发计划表。

## 触发时机
所有输入物已满足，必须先输出《开发计划表》。

## 输入
- 设计文档
- 测试文档
- 任务清单
- 语言代码规范（languages/*.md）

## 输出
```markdown
## 开发计划表

| 任务编号 | 任务名称 | 依赖关系 | 工时(小时) | 风险 | 检查点 |
|---------|---------|---------|-----------|------|--------|
| Task 1 | 创建{资源}API | 无 | 2 | 无 | ✓ |
| Task 2 | 查询{资源}API | Task 1 | 1 | 无 | ✓ |
| Task 3 | 更新{资源}API | Task 2 | 1 | 无 | ✓ |

### 技术栈
- 语言: {语言}
- 框架: {框架}
- 版本: {版本}

### 检查点: After Tasks 1-3
- [ ] 所有测试通过
- [ ] 应用构建成功
- [ ] 核心用户流程端到端可用
```

## 执行步骤

```
1. [PMC-1] 读取设计文档，理解API契约
2. [PMC-2] 读取测试文档，理解验收标准
3. [PMC-3] 读取对应语言的代码规范
4. [PMC-4] 识别与现有代码的集成点
5. [PMC-5] 构建依赖图（纵向切片优先）
6. [PMC-6] 进行任务分解（垂直切片）
7. [PMC-7] 估算工作量（按小时计）
8. [PMC-8] 识别风险点
9. [PMC-9] 输出《开发计划表》
10. [PMC-10] 设置检查点
```

## 垂直切片原则

### 错误方式（面切片）
- Task 1: 全部数据库schema
- Task 2: 全部API
- Task 3: 全部service层

### 正确方式（垂直切片）
- Task 1: 创建{资源}（schema + API + service + repository）
- Task 2: 查询{资源}（query + API + service + repository）
- Task 3: 更新{资源}（update + API + service + repository）

## 任务估算原则

| 任务规模 | 代码量 | 工时 | 示例 |
|---------|-------|------|------|
| XS | 1-10行 | 0.5小时 | 添加一个字段 |
| S | 10-50行 | 1小时 | 单个API endpoint |
| M | 50-200行 | 2-4小时 | 完整CRUD |
| L | 200-500行 | 4-8小时 | 复杂业务逻辑 |
| XL | >500行 | 8小时+ | 拆分！ |

## 禁止行为
- 禁止在开发计划表输出前开始编码
- 禁止拆分出XL级任务
- 禁止任务项超过10个（拆分工具）
