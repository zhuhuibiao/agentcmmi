# OPP 量化评估

## 用途
量化指标报告。

## 触发时机
交付前。

## 输入
- 源代码
- 测试覆盖率报告
- 圈复杂度报告

## 输出
```markdown
## 性能指标评估报告 (OPP)

### 技术栈
- 语言: {语言}
- 框架: {框架}

### 量化指标
| 指标 | 目标 | 实际 | 状态 |
|------|------|------|------|
| 圈复杂度平均 | ≤5 | 3.2 | ✅ |
| 圈复杂度最大 | ≤10 | 8 | ✅ |
| 函数长度平均 | ≤30行 | 24 | ✅ |
| 函数长度最大 | ≤50行 | 42 | ✅ |
| 测试覆盖率 | ≥80% | 82% | ✅ |

### 技术债
- [无 / 列出发现的潜在问题]
```

## 执行步骤

```
1. [OPP-1] 统计圈复杂度分布
2. [OPP-2] 统计代码行数（按层）
3. [OPP-3] 统计测试覆盖率
4. [OPP-4] 统计重复代码率
5. [OPP-5] 生成《性能指标评估报告》
```

## OPP 量化指标

| 指标 | 目标 | 测量方法 |
|------|------|---------|
| 圈复杂度平均 | ≤ 5 | sonarQube / radon |
| 圈复杂度最大 | ≤ 10 | sonarQube / radon |
| 函数长度平均 | ≤ 30行 | cloc |
| 函数长度最大 | ≤ 50行 | cloc |
| 测试覆盖率 | ≥ 80% | pytest-cov / go test -cover |
| 重复代码 | 0 | sonarqube / duplicated |

## 禁止行为
- 禁止不统计直接输出"无问题"
- 禁止忽略超标指标
