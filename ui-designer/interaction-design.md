# 交互设计 (DAR)

## 用途
操作流程、交互模式。

## 触发时机
信息架构完成。

## 输入
- 信息架构文档
- 现有交互模式

## 输出
```markdown
## 交互设计方案

### 用户操作流程
```
用户进入页面 → 点击按钮 → 发起请求 → 显示结果
                              ↓
                         成功: 显示数据
                              ↓
                         失败: 显示错误
```

### 交互模式选择
| 操作 | 选择方案 | 理由 |
|------|---------|------|
| 列表操作 | 表格+批量操作栏 | 符合现有项目风格 |
| 表单提交 | 即时验证+提交按钮 | 减少用户等待 |
| 详情查看 | 抽屉/模态框/新页面 | 根据内容复杂度选择 |

### 动效规范
| 动效类型 | 规范 | 示例 |
|---------|------|------|
| 页面切换 | 淡入淡出 200ms | - |
| 按钮点击 | 缩放 0.95, 100ms | - |
| 弹窗出现 | scale 0.95→1 + 淡入 | - |
| 列表加载 | 骨架屏 | - |

### 决策记录
| 决策点 | 备选方案 | 选择 | 理由 |
|--------|---------|------|------|
| 详情页方式 | 抽屉 vs 模态框 vs 新页面 | 抽屉 | 保持上下文 |
```

## 执行步骤

```
1. [DAR-1] 定义核心用户操作流程
2. [DAR-2] 选择合适的交互模式
3. [DAR-3] 定义动效和反馈
4. [DAR-4] 进行设计方案决策（多方案比较）
5. [DAR-5] 输出《交互设计方案》
```

## 禁止行为
- 禁止选择与项目现有交互模式冲突的方案
