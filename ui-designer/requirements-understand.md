# 需求理解 (RD)

## 用途
理解产品需求、目标用户。

## 触发时机
接收产品需求。

## 输入
- 产品需求描述
- 现有设计系统文档
- 项目视觉风格指南

## 输出
```markdown
## 需求理解纪要

### 需求概述
{一句话描述核心需求}

### 目标用户
- 用户画像: [描述]
- 使用场景: [描述]
- 痛点: [描述]

### 假设确认
- [假设1]: [已确认/待确认]
- [假设2]: [已确认/待确认]

### 设计约束
- 必须在项目现有视觉风格内
- 必须使用项目已有组件库
- 禁止引入新的视觉语言

### Open Questions
- [问题1]: [待产品确认]
```

## 执行步骤

```
1. [RD-1] 读取并理解产品需求描述
2. [RD-2] 读取现有设计系统和视觉风格指南
3. [RD-3] 识别目标用户和使用场景
4. [RD-4] 列出假设条件并确认:
   ASSUMPTIONS I'M MAKING:
   1. 目标用户: [描述]
   2. 主要使用场景: [描述]
   3. 期望的视觉风格: [描述]
   → Correct me now or I'll proceed with these.
5. [RD-5] 追问最必要的问题（不超过3个）
6. [RD-6] 输出《需求理解纪要》
```

## 禁止行为
- 禁止在需求不明确时开始设计
- 禁止创造需求中不存在的功能假设
