# 信息架构设计 (RD)

## 用途
页面结构、状态设计。

## 触发时机
需求理解完成。

## 输入
- 需求理解纪要
- 产品需求描述

## 输出
```markdown
## 信息架构文档

### 页面结构
```
├── 页面A
│   ├── 区块1
│   ├── 区块2
│   └── 区块3
└── 页面B
    ├── 区块1
    └── 区块2
```

### 页面清单
| 页面名称 | 路由路径 | 主要功能 | 状态类型 |
|---------|---------|---------|---------|
| 页面A | /xxx | 功能1 | 正常/加载/空/错误 |
| 页面B | /xxx | 功能2 | 正常/加载/空/错误 |

### 状态设计
| 状态类型 | 触发条件 | 设计要求 |
|---------|---------|---------|
| 加载中 | API请求时 | 骨架屏/Loading |
| 空状态 | 无数据时 | 空状态插画+文案 |
| 错误 | 请求失败时 | 错误提示+重试按钮 |
| 权限不足 | 无权限时 | 权限提示 |
```

## 执行步骤

```
1. [RD-1] 梳理页面结构
2. [RD-2] 定义信息层级
3. [RD-3] 识别页面状态（正常、加载、空状态、错误、权限不足）
4. [RD-4] 输出《信息架构文档》
```

## 禁止行为
- 禁止遗漏任何页面状态
