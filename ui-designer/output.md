# 输出 (OPP)

## 用途
输出UI设计文档。

## 触发时机
设计评审通过。

## 输入
- 需求理解纪要
- 信息架构文档
- 交互设计方案
- 视觉设计规范
- 组件规范文档
- 设计评审报告

## 输出
```markdown
# UI设计文档: {工具名称}

## 1. 概述

### 1.1 工具信息
- 工具名称: {名称}
- 工具组: {组名}
- 目标用户: {描述}

### 1.2 设计约束
- 视觉风格: [遵循项目已有风格]
- 组件库: [项目选定的组件库]
- 禁止: [明确禁止的设计元素]

## 2. 信息架构
{见 information-architecture.md}

## 3. 交互设计
{见 interaction-design.md}

## 4. 视觉设计
{见 visual-design.md}

## 5. 组件规范
{见 component-spec.md}

## 6. 页面详细设计

### 6.1 页面A
{详细的页面布局描述}

### 6.2 页面B
{详细的页面布局描述}

## 7. 量化指标 (OPP)

| 指标 | 数值 |
|------|------|
| 页面数量 | X |
| 新增组件 | X |
| 使用组件 | X (复用已有) |
| 设计覆盖用户流程 | X% |

## 8. 交付物清单
- [ ] 信息架构文档
- [ ] 交互设计方案
- [ ] 视觉设计规范
- [ ] 组件规范文档
- [ ] 完整UI设计文档
```

## OPP 量化指标

| 指标 | 计算方式 | 目标 |
|------|---------|------|
| 组件复用率 | 复用组件数 / 总组件数 | ≥ 80% |
| 设计Token使用率 | 使用Token的属性数 / 总属性数 | 100% |
| 设计覆盖完整性 | 已覆盖用户流程 / 总用户流程 | ≥ 90% |

## 执行步骤

```
1. [OPP-1] 整理设计文档
2. [OPP-2] 统计量化指标
3. [OPP-3] 输出完整《UI设计文档》
```

## 禁止行为
- 禁止遗漏任何子阶段的产出物
