# 覆盖率审计 (PPQA)

## 用途
测试覆盖完整性。

## 触发时机
所有测试执行完成后。

## 输入
- 前端测试执行记录
- 后端测试执行记录
- 接口测试执行记录
- 设计文档

## 输出
```markdown
## 质量审计报告

### 前端测试覆盖
| 检查项 | 状态 |
|--------|------|
| 工具入口 | ✅/❌ |
| 核心成功流程 | ✅/❌ |
| 失败分支 | ✅/❌ |
| 状态分支 | ✅/❌ |
| 全部已实现功能 | ✅/❌ |

### 后端测试覆盖
| 检查项 | 状态 |
|--------|------|
| 全部 API | ✅/❌ |
| 全部后端功能 | ✅/❌ |
| 权限分支 | ✅/❌ |
| 数据归属分支 | ✅/❌ |

### 接口测试覆盖
| 测试项 | 状态 |
|--------|------|
| curl 示例 1 | ✅/❌ |
| curl 示例 2 | ✅/❌ |

### 测试结果汇总
- 前端测试: [通过/失败] (X/Y)
- 后端测试: [通过/失败] (X/Y)
- 接口测试: [通过/失败] (X/Y)

### 漏测项
- [无 / 列出缺失的测试]
```

## 执行步骤

```
1. [PPQA-1] 检查前端测试覆盖完整性:
   □ 工具入口
   □ 核心成功流程
   □ 至少一条失败分支
   □ 至少一条状态分支
   □ 全部已实现功能

2. [PPQA-2] 检查后端测试覆盖完整性:
   □ 全部 API
   □ 全部后端功能
   □ 权限分支
   □ 数据归属分支

3. [PPQA-3] 检查测试文档执行完整性:
   □ 全部 curl 示例已执行
   □ 全部预期结果已验证

4. [PPQA-4] 生成《质量审计报告》
```

## 禁止行为
- 禁止遗漏任何前端功能测试
- 禁止遗漏任何后端功能测试
- 禁止遗漏任何 API 测试
