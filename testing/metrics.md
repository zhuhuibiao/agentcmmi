# 量化评估 (OPP)

## 用途
OPP指标报告。

## 触发时机
测试完成，准备输出验收报告前。

## 输入
- 前端测试执行记录
- 后端测试执行记录
- 接口测试执行记录
- 覆盖率审计报告

## 输出
```markdown
## 性能指标评估报告 (OPP)

### 测试覆盖率
| 类型 | 目标 | 实际 | 状态 |
|------|------|------|------|
| 前端测试覆盖率 | ≥80% | 85% | ✅ |
| 后端测试覆盖率 | ≥80% | 82% | ✅ |
| 接口测试覆盖率 | 100% | 100% | ✅ |

### 测试执行统计
| 类型 | 总数 | 通过 | 失败 |
|------|------|------|------|
| 前端测试 | X | X | X |
| 后端测试 | X | X | X |
| 接口测试 | X | X | X |

### 结论
- 验收通过 / 验收不通过
```

## OPP 量化指标

| 指标 | 目标 |
|------|------|
| 前端测试覆盖率 | ≥ 80% |
| 后端测试覆盖率 | ≥ 80% |
| 接口测试覆盖率 | 100% |
| 测试通过率 | 100% |

## 执行步骤

```
1. [OPP-1] 统计前端测试覆盖率
2. [OPP-2] 统计后端测试覆盖率
3. [OPP-3] 统计接口测试覆盖率
4. [OPP-4] 生成《性能指标评估报告》
5. [OPP-5] 生成《验收报告》
```

## 验收门禁

**验收通过必须满足:**
- [ ] 前端测试全部通过
- [ ] 后端测试全部通过
- [ ] 接口测试全部通过
- [ ] 测试覆盖率达标（≥80%）
- [ ] 无关键缺陷未解决

**任一关键测试失败时:**
- [ ] 必须明确输出失败项
- [ ] 必须输出失败原因
- [ ] 必须拒绝验收通过

## 禁止行为
- 禁止在关键测试失败时判定验收通过
